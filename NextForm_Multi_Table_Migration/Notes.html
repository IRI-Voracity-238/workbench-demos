Migrate Multiple Tables from One Database to Another Database

NextForm is a data migration and reformatting package that helps you convert and re-use the data stored in legacy fields, file formats, and databases.

This project demonstrates NextForm's capability of migrating multiple tables from MySQL to Oracle database using the Multi-Table Migration Job wizard.

IMPORTANT: Before running this demo, execute Reset_demo.sql in the NextForm_Multi_Table_Migration project folder. If it exists, delete the folder named new_batch.

To perform this demo:

  1. Select the NextForm_Multi_Table_Migration folder in the Project Explorer view.
  2. Open the "New Multi Table Migration Job" wizard from the NextForm Menu on the navigation bar.
  3. In the Loader field, select the loader type Oracle - sqlldr, and then click Next.
  4. In the Connection profile field, select the connection of the source tables - MySQL.
  5. In the Tables list, select the tables customers_nf and orders_nf.
  6. The next page, Field Modification Rules, is used for transforming the tables, when desired. Click Next to continue.
  7. On the Data Loading Stage page, select the connection profile of the target location - Oracle.
  8. In the Schema name field, select the SCOTT schema if it is not already selected.
  9. The Summary page shows the extractor, if the tables exist and if not, for which tables a DDL will be created, and the loader information. Click Finish.

After you have completed the job in the wizard:

  1. Open the demo Project Explorer folder to examine all the files created for Extract, Transform and Load. Any of these are available for editing, especially the SortCL scripts, for additional data transformation, reporting and/or protection, in addition to the load target.
  2. Right-click on batch file and select Run As -> Batch Program.
  3. Verify results in the target database tables in the Data Source Explorer.
  4. When done with the demo, delete the folder new_batch.
