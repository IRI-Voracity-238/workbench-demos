Dark Data Discovery

This demonstrates the use of the Dark Data Discovery wizard. This wizard searches information with specified formats based on criteria consisting of patterns, expressions, or string literals. You can enter a new pattern, select one from a pattern library, or create a new pattern to be stored in the library. A list of common patterns is available. The results of the search are organized into a usable format and a metadata file (ddf) can be created. If you include xls and/or xlsx document types in the search, an Excel® Interchange file (.eif), which can be opened and manipulated in Excel using CellShield®, is also created.

This demo uses the IRI Workbench.

Before starting this demo, place all the documents to be searched in a single folder on your computer. The files for this demo are located in:
C:\Documents\data

Recommended steps to perform the demo in the Workbench:

  1. Open the "New Dark Data Discovery" wizard from the Data Discovery Menu.
  2. In the Folder field, make sure Dark Data Discovery is the selected project folder.
  3. Select the check box to create a new folder using the name in the next field. The output files will be put into this folder.
  4. In the Name field, leave the default name of search_output for this example.
  5. Select the check box to create a DDF. This file will be put in the metadata folder.
  6. Click Next to continue to the next page.
  7. In the Search directories section, select the check box for My Computer since the files to be searched reside on this computer.
  8. In the File type extensions section, select the check boxes for all the extensions, or click Select All.
  9. Click Next to continue to the next page.
  10. In the Matcher name field, enter CCN. This is used as the field name in the ddf.
  11. In the Search pattern/Keyword field, enter the following regular expression for credit card numbers:
    \b\d{4}-\d{4}-\d{4}-\d{4}\b
  12. Click Add to table to add the header and expression to the table below.
  13. Return to the Matcher name field to add the next search item: SSN.
  14. In the Search pattern/keyword field, enter the following regular expression for social security numbers:
    \b\d{3}-\d{2}-\d{4}\b
  15. Click Add to table..
  16. Return again to the Matcher name field to add the next search item: Phone.
  17. In the Search pattern/keyword field, enter the following regular expression for telephone numbers:
    \b(?:(?:\+?(1)(?:[ .-]?)?)?(?:\(?([2-9]\d{2})\)?(?:[ .-]?)?)([2-9]\d{2})(?:[ .-]?)(\d{4})(?:[ #eExXtT]*)(\d{2,6})?)\b
  18. Click Add to table..

NOTE: To find and use regular expressions in the pattern library, click Browse in the Search pattern/keyword field.
To create and save a regular expression in the pattern library, click Create.
To select a set file for a values search, click Values Search.

  1. In the Match Options field, select Any Pattern to include all the matches found for each of the patterns listed, or select Specified Order to combine the patterns into a regular expression and include the exact matches of the combined expression.
  2. In the Delimiter field, enter: \t
  3. In the Number of results to preview field, enter: 20
  4. In the Metadata display field, select the type of information to be displayed in the report. For this example, select all.
  5. Click Next. The preview page displays
  6. View the preview page to see that results of the search returned the credit card numbers, social security numbers, and phone numbers, and provided the information selected.
  7. Click Finish.
